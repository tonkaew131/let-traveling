# Project Pitching

# 1. Problem Statement

## 1.1 ปัญหาหรือความต้องการที่ต้องการแก้ไข

ในการวางแผนเที่ยวแบบเดิมต้องสลับไปมาหลายแอปเพื่อเช็กเที่ยวบิน, โรงแรม, สภาพอากาศ และจัดตารางเวลา ทำให้เสียเวลาและยากต่อการคุมงบประมาณ

## 1.2 กลุ่มผู้ใช้งานเป้าหมาย

นักท่องเที่ยวที่ต้องการแผนการเดินทางเป็นของตัวเอง แต่ไม่มีเวลาหาข้อมูลเองทั้งหมด

## 1.3 ทําไมต้องใช้ Generative AI

เพราะระบบแบบเดิมไม่สามารถเข้าใจเงื่อนไขที่ซับซ้อนและความต้องการที่ยืดหยุ่นได้ จัดตารางที่เดินไม่เยอะเพราะพาผู้สูงอายุไป AI จะทำหน้าที่เป็น **Reasoning Engine** ที่เปลี่ยนความต้องการเหล่านี้ให้เป็นแผนที่ใช้งานได้จริงผ่านการเรียกใช้ Tool ต่างๆ

**Generative AI (LLM)** สามารถเข้าใจบริบทภาษา และความต้องการที่เป็นภาษามนุษย์จริงๆ สามารถทำ **Reasoning** เพื่อเลือกข้อมูลที่เหมาะสมจากหลายแหล่งมาสรุปเป็นแผนที่อ่านง่ายในที่เดียว แทนที่ User จะต้องไปไล่กดฟิลเตอร์ในหลายๆ เว็บ

# 2. Proposed Solution

## 2.1 แนวคิดของระบบ

**Let’s** **Traveling** เป็นผู้ช่วยวางแผนการท่องเที่ยวที่ครบวงจร ตั้งแต่การเลือกสถานที่ท่องเที่ยว เช็กสภาพอากาศ คำนวณเวลาเดินทาง จองตั่วเครื่องบินและโรงแรม และคุมงบประมาณให้เหมาะสม โดยผู้ใช้งานสามารถดาวน์โหลด pdf เพื่อนำแผนการเดินทางไปใช้งานได้อย่างสะดวกสบาย

## 2.2 บทบาทของ LLM / Agentic AI / AI Agent ในระบบ

- LLM ทำหน้าที่เป็นตัวกลางในการตัดสินใจว่าคำถามของ User จำเป็นต้องใช้ข้อมูลอะไรบ้าง
- หากข้อมูลที่มีไม่เพียงพอ Agent จะเรียกใช้ API ภายนอกอัตโนมัติ (เช่น เรียก OpenWeather เพื่อดูว่าวันนั้นควรจัดแผนกลางแจ้งหรือไม่)
- ใช้ **PostgreSQL + pgVector** จดจำความต้องการเดิมของ User ใน Chat เพื่อนำมาปรับปรุงแผนในฝั่ง Itinerary ได้แบบ Real-time

## 2.3 จุดเด่นหรือความแตกต่างจากระบบทั่วไป

- ผลลัพธ์ไม่ได้ออกมาเป็นแค่ข้อความ แต่เป็น UI ที่รวมข้อมูล Weather, Booking และ Map ไว้ในหน้าจอเดียว (Dashboard) โดยที่ข้อมูลทุกอย่างสอดคล้องกัน (เช่น ถ้าตั๋วเครื่องบินลงจอดช้า แผนเที่ยววันแรกจะขยับให้อัตโนมัติ)
- User สามารถสั่งเปลี่ยนแผนผ่าน Chat ได้เลย ไม่ต้องเริ่มกรอกฟอร์มใหม่ตั้งแต่ต้น
- มีระบบ **Export PDF** ที่สวยงามผ่าน Puppeteer พร้อมใช้งานทันที

ระบบอื่นที่คล้าย Layla, Wonderplan, Trip Planner, Kayak,แชทบอท AI ทั่วไป

## 3.2 Workflow การทํางานแบบ step-by-step

1. User พิมพ์ความต้องการในหน้าแรก
2. Agent (LLM)  วิเคราะห์คำขอ และตัดสินใจเลือก Tools (Duffel, OpenWeather, Google Places) แบบขนานหรือแบบลำดับตามความจำเป็น
3. ข้อมูลถูกบันทึกลงฐานข้อมูล(PostgreSQL) และLLM สรุปเป็นแผนการเดินทางส่งกลับมาแสดงผลที่หน้า Dashboard
4. User สามารถ แชทแก้แผนการเดินทางได้ทันที เช่น "ขอเปลี่ยนโรงแรม" ซึ่ง Agent จะไปดึงข้อมูลใหม่มาอัปเดต Dashboard เฉพาะส่วน

# 4. Feasibility & Expected Outcome

## 4.1 เทคโนโลยีที่ใช้

- Model Provider; Google AI Studio, OpenRouter API
- Models ที่ใช้; Gemini 3 Flash, Qwen3.5 Plus

    - Tanstack Start สำหรับสร้างเว็บ
    - DrizzleORM สำหรับติดต่อกับฐานข้อมูล
    - TailwindCSS และ ShadCN/ui
